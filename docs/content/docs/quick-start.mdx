# Quick Start

Get your Layer0 development environment up and running in just a few minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 18+** - [Download here](https://nodejs.org/)
- **pnpm** - Install with `npm install -g pnpm`
- **MongoDB** - [Download here](https://www.mongodb.com/try/download/community) or use [MongoDB Atlas](https://www.mongodb.com/atlas)

<Callout type="info">
  This starter kit uses pnpm as the package manager for better performance and
  disk efficiency.
</Callout>

## 1. Clone and Setup

```bash
# Clone the repository
git clone <your-layer0-repo-url>
cd layer0

# Install dependencies for all workspaces
pnpm install
```

## 2. Environment Configuration

### Backend Environment

Create your backend environment file:

```bash
cd backend
cp .env.example .env
```

Edit `backend/.env` with your configuration:

```bash title="backend/.env"
# Authentication
BETTER_AUTH_SECRET=your-secret-key-change-this-in-production-layer0-auth-2024
BETTER_AUTH_URL=http://localhost:3001

# Database
MONGODB_URI=mongodb://localhost:27017/layer0-auth

# Server
PORT=3001

# AWS S3 (Optional - for file uploads)
AWS_ACCESS_KEY_ID=your-access-key
AWS_SECRET_ACCESS_KEY=your-secret-key
AWS_REGION=us-east-1
S3_BUCKET_NAME=your-bucket-name

# Convex (for real-time features)
CONVEX_DEPLOYMENT=your-convex-deployment-url
```

<Callout type="warn">
  **Security**: Make sure to change the `BETTER_AUTH_SECRET` to a strong, unique
  value in production!
</Callout>

### Frontend Environment (Optional)

If you need to customize the backend URL, create `frontend/.env.local`:

```bash title="frontend/.env.local"
NEXT_PUBLIC_API_URL=http://localhost:3001
```

## 3. Start Development Servers

Open two terminal windows and run:

**Terminal 1 - Backend:**

```bash
cd backend
pnpm dev
```

**Terminal 2 - Frontend:**

```bash
cd frontend
pnpm dev
```

**Terminal 3 - Convex (Optional):**

```bash
cd backend
pnpm convex
```

## 4. Access Your Application

- **Frontend**: [http://localhost:3000](http://localhost:3000)
- **Backend API**: [http://localhost:3001](http://localhost:3001)
- **Auth Endpoints**: [http://localhost:3001/api/auth/\*](http://localhost:3001/api/auth/)

## 5. First Steps

1. **Navigate to the app**: Open [http://localhost:3000](http://localhost:3000)
2. **Create an account**: Go to `/auth` and sign up
3. **Explore features**: Check out the dashboard and other protected routes

## Troubleshooting

### Common Issues

**MongoDB Connection Error**

```
Error: MongoNetworkError: connect ECONNREFUSED
```

**Solution**: Ensure MongoDB is running locally or check your `MONGODB_URI`

**CORS Error**

```
Access to fetch at 'http://localhost:3001' from origin 'http://localhost:3000' has been blocked by CORS policy
```

**Solution**: Check that frontend URL is in `trustedOrigins` in the auth config

**Port Already in Use**

```
Error: listen EADDRINUSE :::3000
```

**Solution**: Kill the process using the port or change the port in your configuration

For more troubleshooting help, see our [Troubleshooting Guide](/docs/troubleshooting).
